Systemy BSD były pionierami w wykorzystaniu narzędzi do zarządzania kodem. Z czasem te narzędzia ulegały zmianom i migrowały na alternatywne rozwiązania.

UWAGA: Nie korzystaj z zasobów serwerów macierzystych, zawsze korzystaj z mirrora!</strong>

FreeBSD</strong> obecnie wykorzystuje Subversion i oferuje oficjalny mirror w gicie na GitHubie.

svn:
https://www.freebsd.org/doc/handbook/svn.html</a>

git:
git clone https://github.com/freebsd/freebsd.git</code>

DragonflyBSD</strong> prowadzi rozwój w gicie.

git clone git://git.dragonflybsd.org/dragonfly.git dragonfly</code>

NetBSD</strong> od początku wykorzystuje CVS, mirrory w gicie można znaleźć na GitHubie.

https://www.netbsd.org/docs/current/</a>

CVS:
cvs -d anoncvs@anoncvs.netbsd.org:/cvsroot checkout -P src</code>
cvs -d anoncvs@anoncvs.netbsd.org:/cvsroot checkout -P xsrc</code>
cvs -d anoncvs@anoncvs.netbsd.org:/cvsroot checkout -P pkgsrc</code>

CVS + rsync:
Najpierw należy pobrać repozytoria rsyncem
for m in CVSROOT htdocs src xsrc pkgsrc othersrc; do rsync -az --delete -h --progress rsync://anoncvs.netbsd.org/cvsroot/$m/ $m; done</code>
rsync -az --delete -h --progress rsync://pkgsrc-wip.cvs.sourceforge.net/cvsroot/pkgsrc-wip/ pkgsrc-wip</code>

Następnie w innym katalogu:
cvs -d /ścieżka/do/katalogu/z/CVSROOT -R checkout -P src</code>
cvs -d /ścieżka/do/katalogu/z/CVSROOT -R checkout -P xsrc</code>
cvs -d /ścieżka/do/katalogu/z/CVSROOT -R checkout -P pkgsrc</code>

Mirrory git:
IIJ:
git clone https://github.com/IIJ-NetBSD/netbsd-src.git</code>
Joerg Sonnenberg:
git clone https://github.com/jsonn/src.git</code>
git clone https://github.com/jsonn/xsrc.git</code>
git clone https://github.com/jsonn/pkgsrc.git</code>
Joyent:
git clone https://github.com/joyent/pkgsrc.git</code>
git clone https://github.com/joyent/pkgsrc-wip.git</code>

OpenBSD</strong> podobnie jak NetBSD, od początku swego istnienia wykorzystuje CVS.

http://www.openbsd.org/anoncvs.html</a>
